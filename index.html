Task Breakdown (Plain English)

Click on any task to see a simple explanation of what it means for our team.

1. Data Ingestion & Storage +
What this means: We need to set up the systems that collect the raw data (like live GPS locations from buses or schedules) and save it securely in databases so we can use it later. Think of this as building the plumbing that brings water into the house.
2. Stream & Batch Processing +
What this means: Handling data in two ways. "Stream" processing means dealing with live data the exact second it arrives (to update live maps). "Batch" processing means analyzing large chunks of old data overnight (to find historical traffic patterns).
3. AI/ML ETA Models +
What this means: Using math and algorithms to guess exactly when a bus will arrive. We look at where the bus is now, historical traffic, and other factors to provide an accurate Estimated Time of Arrival (ETA).
4. Prediction API & Analytics +
What this means: Building a digital bridge (an API) so the other groups (like the frontend team building the website/app) can ask our system for the arrival times we calculated, and we send the answers back to them instantly.
5. Data Validation & Quality +
What this means: Setting up automatic checks to make sure the data we receive isn't broken or corrupt. If a GPS tracker sends us a location in the middle of the ocean, this system catches the error before it breaks our predictions.
6. Cross-Group Coordination +
What this means: Talking to the other teams. We need to clearly define what format the hardware/mobile team will send us GPS data in, and what format the frontend team expects us to return the ETA data in.
