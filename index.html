Task Breakdown (Plain English)

Click on any task to see a simple explanation of what it means for our team.

1. Data Ingestion & Storage +
What this means: We set up systems that collect raw data — like live GPS locations from buses and timetable schedules — and store them securely in databases. Think of it as building the plumbing that brings water into the house. Nothing else can work until this is in place.
2. Stream & Batch Processing +
What this means: Handling data in two ways. "Stream" processing deals with live data the moment it arrives — updating a map in real time. "Batch" processing analyzes large amounts of old data at once, like running a nightly report to find recurring traffic patterns.
3. AI/ML ETA Models +
What this means: Using algorithms to predict exactly when a bus will arrive at a stop. The model learns from historical patterns — traffic, time of day, weather — to give commuters a reliable Estimated Time of Arrival (ETA). This is our most important deliverable.
4. Prediction API & Analytics +
What this means: Building a digital interface (an API) that other groups — like the team building the mobile app — can call to get ETA predictions from us. When they ask "when is Bus 47 arriving?", our system answers in under a second.
5. Data Validation & Quality +
What this means: Automatic checks that catch corrupt or impossible data before it breaks our models. If a GPS tracker reports a bus in the ocean, or a timestamp jumps backwards in time, these checks flag it so we never train our AI on bad data.
6. Cross-Group Coordination +
What this means: Talking with other sub-groups to agree on contracts. We need to know: what format will the hardware/GPS team send us data in, and what format does the frontend team expect us to return predictions in? This prevents integration failures later.
