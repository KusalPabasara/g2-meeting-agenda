🏗 High-Level System Architecture 15 min
Sketch the data flow end-to-end within G2's scope
▼
Architecture Layers to Discuss
  • Ingestion Layer: Kafka topics for GPS events, REST/WebSocket connectors for live feeds
  • Stream Processing: Flink / Spark Streaming for real-time ETA computation
  • Batch Layer: Spark batch jobs + Airflow DAGs for model training data prep
  • Storage: InfluxDB (time-series GPS), PostgreSQL (routes, schedules, users)
  • ML Layer: model training (PyTorch/TF) → MLflow tracking → model registry
  • Serving Layer: FastAPI endpoints for ETA predictions & analytics queries
  • Quality Layer: Great Expectations checks at ingestion and post-processing
Key Questions to Resolve
  • Monorepo vs. separate repos per component?
  • Containerisation strategy — Docker Compose for local, Kubernetes for prod?
  • Data schema design — agree on GPS event payload format
  • Latency SLA: what is an acceptable ETA prediction delay? (target: <2s?)
